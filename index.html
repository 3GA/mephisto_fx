Mephisto III

A raspberry pi based internet radio with a Java FX UI

Download this project as a .zip file Download this project as a tar.gz file

Motivation

This is the third internet radio I've build. It has been mostly been inspired by this radio that has been published on hackaday.com. The idea of attaching a cheap display to the raspi for a radio made me thinking what kind of other information could be displayed. Since I already developed a Google Music client for my last radio, I decided to build another one. Also, since I am mostly using the radio in the morning, I decided to add weather and time information to the UI too. Have a look on the results below!

The Hardware

Bill of materials:

  • Wood box
  • Raspberry Pi
  • 3x USB hub
  • USB WLAN stick
  • USB sound stick
  • 2x Visaton FRS8 speakers
  • 1x rotary encoder with push
  • 2x relays
  • 4,3 Zoll TFT disply (look for TFTs for cars)
  • 1x component cable
  • 40x 20cm male - female jumper wire cable (connecting the rotary encoder with the raspi)
  • 1x micro USB cable (I've cut this cable to use one half for the raspi power supply and the other one to connect the first relay)
  • plugs for the power supply

The hardware architecture of the radio is pretty simple. The raspberry pi with USB WLAN and USB sound has another USB cable attached that will power a relay. This relay enables the power support for the Logitech amp and another relay that enables the display. I used the switch of the poti of the Logitech amp to control the power to the raspberry pi. So nothing fancy here ;)

The Software

The main motivation for this radio was to build a UI using JavaFX. There are a lot of demos out there that show that JavaFX is working on the pi, but I did not find much use cases. I wanted to have a nice UI and it took most of the time to build it. I haven't programmed JavaFX before and the functionality of the API is really impressing. Unfortunatly the performance of the raspi is not sufficient for all effects to run smoothly, especially when other thread are running (e.g. threads that load the UI for the Google music section that is loaded asynchronously). I would have have switched to a beaglebone black board, but unfortunatly the board does not provide a component out signal.

Additional software used:

  • Gaugette's rotary encoder library
  • mpc/mpd for music playback
  • A small python programm that communicates the rotary encoder state with the JavaFX programm via a socket

Conclusion

Programming JavaFX was fun, also to actual "build" something. My next project is gonna be a gainclone.
